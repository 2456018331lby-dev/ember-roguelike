# 《余烬》(Ember) — 完整游戏设计文档

> 一款融合卡牌构筑的双模式肉鸽手游，每一局都是独一无二的抉择

---

## 一、游戏概述

### 核心理念
**"一切皆可交换，死亡皆有意义"**

《余烬》是一个双模式肉鸽游戏，共享卡牌底层系统，两种截然不同的游戏体验：
- **献祭模式**：你就是自己，每一次变强都要舍弃什么
- **寄生模式**：你不是英雄，你是寄生在英雄身上的东西

### 目标平台
- 第一阶段：Android（触屏原生）
- 第二阶段：iOS / PC (Steam)

### 技术栈
- **引擎**: Godot 4.x (GDScript)
- **理由**: 开源免费、2D 性能优秀、导出 Android 一键搞定、跨平台一致
- **美术风格**: 像素风 16-32px，暗色调为主，强调氛围
- **音效**: 8-bit/Chiptune 风格，紧张氛围配乐

### 目标用户
- 15-35 岁，喜欢肉鸽/策略/卡牌的玩家
- 碎片时间玩家（一局 10-15 分钟）
- 追求 build 极限的硬核玩家

## 二、共享系统：卡牌引擎

### 2.1 卡牌基础

所有模式共享同一套卡牌系统。卡牌是能力的载体。

**卡牌类型**：
| 类型 | 说明 | 触发方式 |
|------|------|---------|
| 攻击牌 | 造成伤害 | 主动打出 |
| 防御牌 | 提供护盾/闪避 | 主动打出 |
| 被动牌 | 持续效果 | 放入槽位自动生效 |
| 诅咒牌 | 负面效果 | 强制获得，不可丢弃（除非特殊手段） |
| 融合牌 | 两张牌合成 | 献祭/寄生专属触发 |

**卡牌稀有度**：
- 白色（普通）— 基础效果
- 绿色（精良）— 中等效果，附带小条件
- 蓝色（稀有）— 强力效果，附带代价
- 紫色（史诗）— 改变规则的效果
- 金色（传说）— 极其稀有，彻底改变玩法

### 2.2 小丑牌系统 (灵感来源: Balatro)

**核心概念：小丑牌 = 全局 Modifier**

不同于普通攻击/防御牌，小丑牌是"改变规则"的牌。最多装备 5 张。

示例小丑牌：
```
【血债】- 每次受伤时，下次攻击伤害 x1.5（可叠加）
【镜像】- 你的攻击同时命中自己（但伤害 x2）
【债主】- 每波结束时失去 10% 生命，但商店价格减半
【赌徒】- 每张牌 50% 概率效果翻倍，50% 概率无效
【收藏家】- 每拥有 10 张牌，所有牌效果 +5%
【倒计时】- 战斗开始时倒计 10 秒，到 0 时秒杀所有敌人
【怜悯】- 敌人血量低于 10% 时自动死亡，但你无法获得击杀奖励
```

小丑牌之间可以产生**连锁反应**：
```
【血债】+【镜像】= 自伤触发血债叠加，攻击时效果爆炸
【赌徒】+【收藏家】= 牌越多越容易触发翻倍
【债主】+【倒计时】= 低血量但秒杀，高风险高回报
```

### 2.3 牌组管理

- **手牌上限**: 7 张（战斗中抽到上限停止抽牌）
- **每回合抽牌**: 3 张（可通过被动牌/小丑牌增加）
- **弃牌**: 每回合可弃 2 张（可通过装备增加）
- **牌库**: 战斗结束后手牌归入牌库，重新洗牌
- **牌组上限**: 30 张（超过时必须销毁一张才能获得新牌）

## 三、模式一：献祭 (Sacrifice)

### 3.1 核心规则

**"想要什么？先放弃什么。"**

游戏里没有免费的强化。每一次变强都需要牺牲一样东西。

**战斗框架（类 Brotato）**：
- 俯视角竞技场，自动攻击
- 左手虚拟摇杆移动，右手滑动瞄准/释放技能
- 每波 20-30 秒，共 20 波
- 波间进入商店/献祭界面
- 一局约 10-15 分钟

### 3.2 献祭系统

**基础献祭**：
每次在商店/奖励界面，你获得的卡牌都附带一个"献祭代价"：

```
获得【烈焰剑】(攻击牌)
代价：永久失去 15% 移速

获得【铁壁】(被动牌)
代价：永久失去 20% 攻速

获得【血之契约】(小丑牌)
代价：每波开始时失去 5% 最大生命值
```

**连锁献祭**：
连续牺牲同一类属性会触发"极端化"效果：

| 牺牲次数 | 效果 | 名称 |
|---------|------|------|
| 3次移速 | 无法移动，但伤害 x3 | 磐石之躯 |
| 3次攻速 | 攻击间隔极长，但单发伤害爆炸 | 狙击手 |
| 3次生命 | 最大生命锁定为 1，但有 5 秒无敌 | 幽灵 |
| 3次攻击 | 无法攻击，但被动/诅咒效果 x5 | 诅咒之王 |

**隐藏配方**（社区驱动）：
特定的献祭组合触发隐藏效果，游戏内不提示，靠玩家发现和分享：
```
牺牲移速 + 牺牲攻速 + 获得【龟壳】→ 触发"时间冻结"（敌人减速 80%）
牺牲生命 x2 + 获得【亡灵之触】→ 触发"亡者归来"（死亡时复活一次，全属性+50%）
```

### 3.3 献祭模式的卡牌联动

卡牌不仅有正面效果，还可以"献祭卡牌"本身：
- 献祭一张攻击牌 → 获得攻击力永久 +5
- 献祭一张被动牌 → 获得一个被动槽位
- 献祭一张小丑牌 → 获得一次"免费升级"（不需献祭属性）

**核心决策**："我是用这张牌，还是献祭它换永久收益？"

### 3.4 Meta 进度

- **遗物碎片**：死亡后，你献祭过的东西化为碎片
- 碎片可在下一局开始时携带（最多 2 个），相当于"上次的牺牲回报了"
- 碎片合成永久天赋（但合成本身也需献祭其他碎片）
- **献祭图鉴**：记录所有发现过的献祭组合和隐藏配方

## 四、模式二：寄生 (Parasite)

### 4.1 核心规则

**"你不是英雄，你是英雄身上的东西。"**

你是一个寄生体，没有自己的身体。你只能借别人的身体战斗。

**战斗框架（类以撒）**：
- 俯视角地牢探索，房间制
- 摇杆移动 + 攻击/技能按钮
- 每层 8-12 个房间，5-6 层一局
- 一局约 10-15 分钟

### 4.2 宿主系统

**宿主种类**：
| 宿主 | 基础属性 | 特殊能力 | 卡牌倾向 |
|------|---------|---------|---------|
| 战士 | 高血量/低速 | 冲刺/格挡 | 攻击牌+防御牌 |
| 法师 | 低血量/中速 | 闪现/元素 | 诅咒牌+融合牌 |
| 野兽 | 中血量/高速 | 撕咬/咆哮 | 被动牌+小丑牌 |
| 盗贼 | 低血量/极高速 | 隐匿/背刺 | 攻击牌+小丑牌 |
| 圣职 | 高血量/低速 | 治愈/净化 | 防御牌+被动牌 |

**宿主死亡时**：
1. 你从尸体中爬出来 → 变成"裸寄生体"
2. 裸寄生体只有基础移动，无法攻击，持续 5 秒
3. 5 秒内必须寄生新目标（小怪/NPC/宝箱怪）
4. 超时 → 死亡（真正的 Game Over）

**寄生继承**：
新宿主继承上一个宿主的部分卡牌和能力：
- 寄生时间 < 1 层 → 继承 1 张牌
- 寄生时间 1-2 层 → 继承 2 张牌
- 寄生时间 3 层以上 → 继承 3 张牌 + 1 个小丑牌槽位

**寄生融合**：
继承的卡牌和新宿主的卡牌可以触发融合：
```
战士的【旋风斩】+ 法师的【火球术】→ 融合为【火焰旋风】
盗贼的【背刺】+ 野兽的【撕咬】→ 融合为【暗影猎杀】
圣职的【治愈】+ 诅咒牌【血之契约】→ 融合为【生命窃取】
```

### 4.3 寄生模式的卡牌联动

- 每个宿主有独特的"初始牌组"（3-4 张）
- 寄生时继承的牌和新宿主的牌混在一起
- 如果继承牌和新宿主牌产生"共鸣"（同类型 3 张以上），触发额外效果
- **寄生专属小丑牌**：
  ```
  【共生】- 当前宿主每存活 1 分钟，继承比例 +5%
  【蜕皮】- 主动更换宿主时保留全部牌（但新宿主血量减半）
  【吞噬】- 寄生精英怪时，获得其一个特殊技能
  ```

### 4.4 Meta 进度

- **寄生图鉴**：击败新敌人 → 解锁该敌人作为可寄生宿主
- **基因库**：累积经验解锁寄生体被动能力
- **完美寄生**成就：同一宿主通关（极难）
- **宿主收藏**：解锁所有宿主种类

## 五、共享 Meta 系统

### 5.1 余烬之炉（永久进度）

两个模式共享一个"余烬之炉"：
- 每局游戏（无论哪种模式）获得"余烬"
- 余烬用于解锁：
  - 新卡牌（加入全局卡池）
  - 新小丑牌
  - 新宿主（寄生模式）
  - 新献祭选项（献祭模式）
  - 角色外观
  - 难度选项

### 5.2 难度系统

| 难度 | 解锁条件 | 效果 |
|------|---------|------|
| 余烬 I | 默认 | 基础难度 |
| 余烬 II | 通关余烬 I | 敌人+20% 血量，牌价格+15% |
| 余烬 III | 通关余烬 II | 敌人+40%，诅咒牌出现率+25% |
| 余烬 IV | 通关余烬 III | 敌人+60%，每波多 2 个敌人 |
| 余烬 V | 通关余烬 IV | 敌人+100%，宿主初始血量-30% |
| 无尽 | 通关余烬 V | 无上限递增，排行榜竞争 |

### 5.3 每日挑战

- 每天生成一个固定种子的挑战
- 所有玩家用同样的随机种子竞争
- 排行榜按"得分"排序（伤害/存活时间/献祭次数/寄生次数综合）
- 奖励：特殊卡牌/外观

### 6.2 核心系统设计

**GameManager (Autoload)**：
```
- current_mode: enum (SACRIFICE, PARASITE)
- current_run: RunData
- meta_progress: MetaData
- card_pool: CardPool
- state: enum (MENU, PLAYING, SHOP, GAME_OVER)
```

**CardBase (Resource)**：
```
- id: String
- name: String
- type: enum (ATTACK, DEFENSE, PASSIVE, CURSE, JOKER, FUSION)
- rarity: enum (WHITE, GREEN, BLUE, PURPLE, GOLD)
- cost: int (行动点)
- effects: Array[Effect]
- sacrifice_cost: SacrificeCost (献祭模式专用)
- description: String
- icon: Texture2D
```

**SacrificeSystem**：
```
- sacrificed_stats: Dictionary {stat: amount}
- sacrifice_count: Dictionary {stat: count}
- hidden_recipes: Array[Recipe]  # 从 JSON 加载
- check_extreme化() -> 触发极端化效果
- check_hidden_recipe() -> 检查隐藏配方
```

**HostSystem (Parasite)**：
```
- current_host: Host
- inheritance_queue: Array[Card]  # 继承的牌
- inheritance_percent: float
- host_history: Array[Host]  # 历史宿主
- check_fusion(card_a, card_b) -> Card | null
```

### 6.3 触屏操作设计

**献祭模式**：
- 左侧：虚拟摇杆（移动）
- 右侧：滑动手势（释放卡牌技能）
- 底部：手牌区域（滑动出牌）
- 右上：小丑牌槽（点击切换）

**寄生模式**：
- 左侧：虚拟摇杆（移动）
- 右侧：攻击按钮 + 技能按钮
- 底部：手牌区域
- 寄生时：点击目标触发寄生
